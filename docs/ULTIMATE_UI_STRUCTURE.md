## 一、顺时终极 UI 原则

### 1.1 核心定义

顺时不是"功能很多的健康 App"，而是：

> **低注意力、低学习成本、长期可托付的生活系统**

### 1.2 六条铁律

| # | 原则 | 描述 |
|---|------|------|
| 1 | **一眼就知道今天该做什么** | 用户打开后直接看到：今日洞察 + 今日3个小行动 |
| 2 | **内容很多，界面不能复杂** | 内容很深，界面很浅 |
| 3 | **AI是主入口，但不是唯一入口** | 想问→聊，想看→点，想跟进→首页有 |
| 4 | **永远不要让用户"管理自己"** | 不是打卡、评分、指数，而是被轻轻托住 |
| 5 | **首页是"生活状态页"，不是"功能分发页"** | 我现在怎样 + 今天该做什么 + 有什么需要被接住 |
| 6 | **家庭视角和个人视角必须分开** | 对自己温柔细腻不打扰，对子女简洁明确安心 |

## 四、年龄阶段动态首页

### 4.1 18-25岁：轻节奏

```
优先级：节律 > 睡眠 > 情绪 > 运动

内容：
1. 今日节律
2. AI对话入口
3. 睡眠建议
4. 轻运动推荐
```

### 4.2 25-40岁：压力管理

```
优先级：洞察 > 计划 > 跟进 > 节气

内容：
1. 今日洞察
2. 三件小事
3. 跟进卡片
4. AI对话
5. 节气提醒
```

### 4.3 40-60岁：调养

```
优先级：节气 > 体质 > 食疗 > 建议

内容：
1. 节气建议
2. 今日三件小事
3. 体质调养入口
4. 食疗推荐
5. AI对话
```

### 4.4 60+岁：陪伴

```
优先级：简单 > 陪伴 > 语音

特点：
- 字号更大
- 操作更少
- 元素更少
- 语音优先

内容：
1. 今日一句
2. 语音聊天入口
3. 今日建议
4. 节气一句
```

## 六、10年进化路径

### 阶段1：App时代
- 手机端首页 + AI对话 + 节气 + 内容

### 阶段2：家庭时代
- 增加家庭页
- 子女端安心视图

### 阶段3：设备时代
- 手表小组件
- 车机轻提醒
- 智能音箱语音陪伴

### 阶段4：系统时代
- 顺时不再只是App
- 成为用户"生活中的默认节律层"

## 九、设计原则总结

> **顺时的界面，不应该让人感觉"我在使用一个产品"，**
> **而应该让人感觉："我的生活被轻轻整理了一下。"**
